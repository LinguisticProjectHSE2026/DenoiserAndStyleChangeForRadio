# Очистка и смена стиля записей радио

Реализация пайплайна для очистки записей чукотского радио: убираем музыку, разделяем пение и речь, отфильтровываем фрагменты ненужных языков и при необходимости меняем стиль записей.

Записи содержат смесь языков — в основном чукотский и русский. Для каждой записи в `input/` пайплайн:

1. **Предобработка** (`preprocessor/`): demucs убирает фоновую музыку (оставляет только голос), затем аудио режется по тишине, каждый фрагмент классифицируется как `speech` (речь) или `singing` (пение) по коэффициенту вариации высоты тона и раскладывается в `tmp/speech/` и `tmp/singing/`. Фрагменты ненужных языков (например, русского) отбрасываются языковым фильтром (по умолчанию MMS-LID — у него есть класс для чукотского; доступен и Whisper). Дополнительно вырезаются неречевые звуки (вздохи, придыхания) детектором речи Silero VAD.
2. **Смена стиля** (`style_changer/`): собирает фрагменты обратно в один файл на каждую запись в `output/`. По умолчанию **голос сохраняется** и скорость не меняется. Опционально можно замедлить речь для лучшей разборчивости (`STYLE_SPEED`) или заменить голос на чистый целевой с помощью **конверсии голоса** по веткам — `speech` → [kNN-VC], `singing` → [kNN-SVC].

## Установка

```bash
pip install -r requirements.txt   # нужен ffmpeg в PATH
```

## Использование

1. Положите исходные записи в `input/`.
2. Запустите пайплайн:

```bash
python -m pipeline      # пишет output/<имя> для каждого input/<имя>
```

По умолчанию обрабатывается **только речь** (пение всё равно классифицируется в `tmp/singing/`, но в вывод не попадает — определение языка в пении ненадёжно). Чтобы включить пение: `STYLE_LABELS=speech,singing`.

Голос сохраняется, скорость исходная. Чтобы слова было легче разбирать, замедлите запись (с сохранением высоты тона) через `STYLE_SPEED` — например, `STYLE_SPEED=0.85`.

### Опционально: замена голоса на чистый целевой

Чтобы сменить голос на чистый целевой, предоставьте целевое аудио:

```bash
# ~10-20 минут чистой речи на чукотском в target/
STYLE_SPEECH_VC=knn-vc STYLE_SINGING_VC=knn-svc python -m pipeline
```

Целевой голос на чужом языке (например, английский диктор) оставляет «акцент» из чужих фонем на чукотском выводе и ухудшает разборчивость — язык должен совпадать. При первом запуске конверсии веса kNN-VC (WavLM + HiFi-GAN) скачиваются через `torch.hub`.

### Выбор устройства

Устройство выбирается автоматически `cuda → mps → cpu`; переопределяется через `STYLE_DEVICE`.

- **Colab / NVIDIA GPU:** ничего делать не нужно — CUDA выбирается автоматически.
- **Проверка на Mac:** принудительно CPU и включить fallback на MPS для неподдерживаемых операций:

```bash
STYLE_DEVICE=cpu PYTORCH_ENABLE_MPS_FALLBACK=1 python -m pipeline
```

При `STYLE_SINGING_VC=knn-svc` для пения используется [kNN-SVC] (основа та же, WavLM+kNN плюс аддитивный синтез, работает для пения лучше чем kNN-VC). Через pip не ставится, поэтому настраивается один раз:

```bash
git clone https://github.com/SmoothKen/knn-svc          # укажите KNN_SVC_DIR, если он в другом месте
pip install pyworld librosa soundfile                    # доп. зависимости
# скачайте чекпойнт *_mix_harm_no_amp_*.pt из Releases в KNN_SVC_CKPT_DIR
```

Если не настроено, пайплайн один раз предупреждает и откатывается на kNN-VC для пения. Все фрагменты пения конвертируются **одним батчем** (модель и цель загружаются один раз); предпочтителен GPU. Для пения можно также указать референс через `STYLE_SINGING_TARGET_DIR` (по умолчанию берётся `STYLE_TARGET_DIR`).

## Заметки и ограничения

- Вывод — **44.1 кГц моно**. При сохранении голоса исходная полоса частот сохраняется; конверсия голоса работает на 16 кГц моно, затем частота повышается до 44.1 кГц.
- **Паузы (тишина) между фрагментами по умолчанию убираются.** Препроцессор пишет `manifest.json` с таймкодами фрагментов, поэтому `STYLE_RESTORE_GAPS=1` восстанавливает исходные паузы для более естественного темпа.
